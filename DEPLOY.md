# Deployment Guide

WebSAM is deployed to Cloudflare Workers with models on Cloudflare R2.

## Architecture

- **`websam.xevion.dev`** -- SvelteKit app (Cloudflare Worker)
- **`data.websam.xevion.dev`** -- R2 bucket CDN (demo images)
- Models served via presigned S3 URLs (generated by `/api/model-url` route)
- ORT WASM files served via `/wasm/[filename]` proxy route from R2

## GitHub Secrets (Required)

| Secret | Purpose |
|--------|---------|
| `CLOUDFLARE_API_TOKEN` | Wrangler deploy authentication |
| `CLOUDFLARE_ACCOUNT_ID` | Cloudflare account identifier |
| `R2_ACCESS_KEY_ID` | R2 S3-compatible API access key |
| `R2_SECRET_ACCESS_KEY` | R2 S3-compatible API secret key |

## Worker Secrets

The presigned URL route needs R2 credentials at runtime. These are **Worker secrets**,
separate from GitHub secrets. Set them once after first deploy:

```bash
echo "YOUR_KEY" | bunx wrangler secret put R2_ACCESS_KEY_ID
echo "YOUR_SECRET" | bunx wrangler secret put R2_SECRET_ACCESS_KEY
echo "YOUR_ACCOUNT_ID" | bunx wrangler secret put R2_ACCOUNT_ID
```

The deploy workflow also passes these via `wrangler-action`'s `secrets` field,
so subsequent deploys preserve them.

## Initial Setup: Upload to R2

The upload script is idempotent -- it skips files that already exist with matching size.

```bash
# Upload everything (models + WASM + demo images)
bun run scripts/upload.ts

# Specific model only
bun run scripts/upload.ts --model=sam2.1-tiny

# WASM files only
bun run scripts/upload.ts --wasm-only

# Demo images only
bun run scripts/upload.ts --demos-only

# Preview what would be uploaded
bun run scripts/upload.ts --dry-run
```

Credentials are read from `.env` (R2_ACCESS_KEY_ID, R2_SECRET_ACCESS_KEY, R2_ACCOUNT_ID).

## Deployment

Automatic on push to `master` via `.github/workflows/deploy.yml`.

### Manual Deploy

```bash
bun run deploy           # Uses CLOUDFLARE_API_TOKEN from .env
bun run deploy:dry-run   # Preview only
```

**Troubleshooting: "Failed to fetch auth token"**

If wrangler fails with this error, a stale OAuth token may exist at
`~/.config/.wrangler/config/default.toml`. Delete that file -- wrangler
will then use `CLOUDFLARE_API_TOKEN` from `.env` instead.

## CORS for Demo Images

Demo images are served from `data.websam.xevion.dev` (R2 custom domain) with
Cloudflare Image Resizing (`/cdn-cgi/image/...`). Image Resizing does not
pass through R2's CORS headers, so a **Response Header Modification Rule** on
the `xevion.dev` zone is required.

**Cloudflare Dashboard > Rules > Transform Rules > Modify Response Header:**

- **Expression**: `http.host eq "data.websam.xevion.dev"` (combined via `or`
  with any other CDN hostnames sharing the same rule)
- **Action**: Set `Access-Control-Allow-Origin` to `*`

Without this rule, browser fetches from `websam.xevion.dev` to the CDN will
fail with a CORS error.

## Local Development

```bash
bun run dev    # Vite dev server -- models load from local /models dir
```

Local dev does **not** use R2. Models are fetched directly by the browser
from HuggingFace or local cache (via the existing download pipeline).

## Configuration

- **`wrangler.toml`** -- Worker config, R2 bucket binding, custom domain route
- **`.github/workflows/deploy.yml`** -- CI/CD pipeline
- **`VITE_CDN_HOST`** -- GitHub Actions variable for demo image CDN (`https://data.websam.xevion.dev`)
